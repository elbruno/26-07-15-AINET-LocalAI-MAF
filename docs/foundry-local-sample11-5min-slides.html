Foundry Local lifecycle

Runtime lifecycle: discover, download, cache, serve

  • Discover runtime backends: execution providers are discovered and registered on startup.
  • Resolve model by alias: app requests the selected NVIDIA/Nemotron streaming model from the local catalog.
  • Download if missing: first run triggers Foundry-managed download and model load.
  • Reuse cache on later runs: cached model avoids repeated download work and starts faster.
  • Create local endpoint/session: Foundry Local binds runtime services on available local ports and the app opens a live transcription session.
  • Optional cleanup: app can unload and remove model cache to force a fresh first-run path.

Sample 11 code

Key C# code to call out live

EP discovery Model download/load Live stream session Unload + cache cleanup 
await FoundryLocalManager.CreateAsync(config, NullLogger.Instance);
var manager = FoundryLocalManager.Instance;

var executionProviders = manager.DiscoverEps();
await manager.DownloadAndRegisterEpsAsync(...);

var catalog = await manager.GetCatalogAsync();
var model = await catalog.GetModelAsync(modelAlias);
await model.DownloadAsync(...);   // first run download
await model.LoadAsync();

var audioClient = await model.GetAudioClientAsync();
var session = audioClient.CreateLiveTranscriptionSession();
await session.StartAsync(CancellationToken.None);
await foreach (var result in session.GetStream(CancellationToken.None)) { ... }

await loadedModel.UnloadAsync();
await loadedModel.RemoveFromCacheAsync(); // optional
