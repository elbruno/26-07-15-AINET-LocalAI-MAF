5 / 7Zero cloud calls; fully local path

First run vs cached run

  • First run: execution providers + selected model are downloaded/registered, then loaded.
  • Cached run: model resolves from local Foundry cache, so startup is faster.
  • Optional reset: choose cleanup at exit (RemoveFromCacheAsync()) to force a fresh first-run path next time.

Call this out live so the audience understands the startup-time difference.
